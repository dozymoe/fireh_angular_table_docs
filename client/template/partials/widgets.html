Widgets

All packaged widgets are set as attribute directives.

The widgets don't have to be laid in an hierarchical structure. Your <table> can be located in the middle of the page while your filter and sorting widget in the sidebar or somewhere else.

Table related widgets

  • FhTable

    Contrary to what the name implies, you don't need to attach this widget to a <table>, and it doesn't provide any html markup.

    A wrapper to FhTableListResourceControllerMixin, provides scope functions related to selection of all rows.

    See fhTable demo.

  • FhTablePager

    Pager navigation.

    See fhTablePager demo

  • FhTablePagerSize

    Select pager sizes.

    See fhTablePagerSize demo

  • FhTableSorting

    See fhTableSorting demo

  • FhTableFilterText

    Filter table rows by field value.

    See fhTableFilterText demo

  • FhTableRow

    Allows you to interact with the data of each row, be it editing, deletion or other custom activities.

    See fhTableRow demo

  • FhTableFilterSelect

    Filter table rows by multiple selection.

    See fhTableFilterSelect demo

  • FhTableBatchActions

    Perform actions on selected rows.

    See fhTableBatchActions demo

  • FhTranscluded

    When a widget allowed you to set its content by writing sub elements html markup, this is called transclusion.

    You can set other angular directives in the content, but there is a bug, the directives will be initialized twice, an exception to this is when the directives are attached to elements with ng-repeat.

    This is a pseudo directive (there is no angular.directive(), just an html attribute treated as special).

    For example, normally you would: 

    
<div data-widget-with-transclusion>
    <div class="widget-s-content" data-another-directive
            data-another-directive-params="one">
    </div>
</div>

    That, will cause bugs, because of the transclusion thingy. The correct markup: 

    
<div data-widget-with-transclusion>
    <div class="widget-s-content" 
            data-fh-transcluded="data-another-directive"
            data-another-directive-params="one">
    </div>
</div>
