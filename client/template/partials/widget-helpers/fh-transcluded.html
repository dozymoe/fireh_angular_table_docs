FhTranscluded

When a widget allowed you to set its content by writing sub elements html markup, this is called transclusion.

You can set other angular directives in the content, but there is a bug, the directives will be initialized twice, an exception to this is when the directives are attached to elements with ng-repeat.

This is a pseudo directive (there is no angular.directive(), just an html attribute treated as special).

For example, normally you would: 


<div data-widget-with-transclusion>
    <div class="widget-s-content" data-another-directive
            data-another-directive-params="one">
    </div>
</div>

That, will cause bugs, because of the transclusion thingy. The correct markup: 


<div data-widget-with-transclusion>
    <div class="widget-s-content" 
            data-fh-transcluded="data-another-directive"
            data-another-directive-params="one">
    </div>
</div>
