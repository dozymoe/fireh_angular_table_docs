FhCustomEventHandlersMixin

Helps develop your own widget, provides consistent/uniform interface.

Event-hub is the core of this framework. While you would normally interact with the widget by listening or triggering events, there is a method of overloaded event listener that can modify behaviour of the widgets themselves.

With "event handlers" you could gain access to the scope and instance of FhTableDefinition that the widget were using.

While the widget is processing listened events, the mixin will look at the html attribute data-fhe- + event's name. If existed will run the scope function first, before calling the widget's original event listener.

If the attribute was not found, will look at instance of FhTableDefinition for eventHandlers.

It's better if you used the html attribute option, it'd be activated only for the intended widget.

Parameters

  1. eventHandlers

    Object of event's name and array of event handler functions.

  2. attrs

    Angular controller's $attrs.

  3. fhtable

    Instance of FhTableDefinition.

Results

  1. Modify inplace the first argument (eventHandlers), add more callbacks.
