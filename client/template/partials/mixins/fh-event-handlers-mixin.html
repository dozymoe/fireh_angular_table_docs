FhEventHandlersMixin

Help develop your own widget, provides consistent/uniform interface.

Chain event handlers of the same event name together, and install the final event listeners.

Each event handlers is responsible for calling this.nextCallback.

You'd also have access to this.scope, the widget's scope, and this.fhtable, instance of FhTableDefinition the widget is working on.

this.fhtable should also be available as this.scope.fhtable, not sure.

Parameters

  1. eventHandlers

    Object of event's name and array of event handler functions.

  2. options

    The object that you wanted to bind to event-handlers, at minimum they should have these properties: scope, fhtable.

  3. cleanupList (optional)

    List of cleanup callbacks, such as one produced by $scope.$on and $attrs.$observe.
    The list will be modified inplace with cleanup callbacks generated by this mixin.

Results

  1. Return object with the following properties: eventCleanupCallbacks.
  2. Modify the third argument (cleanupList) inplace, the added callback functions will be the same as the ones in the returned object.
    The third argument is added to the api for simplicity.
