FhElementIdMixin

Helps develop your own widget, provides consistent/uniform interface.

Usually applied to form widgets.

Let you use html attributes data-fhp-element-id and data-fhp-element-name with the widget.

The mixin doesn't modify scope.

If you use data-fhp-element-id the generated id is going to be exactly that, but if you use data-fhp-element-name it is going to be random (with some numbers appended), if you didn't set any, a random id will also be generated.

The packaged widgets would usually assign the return value to $scope.elementId and later use it as html attribute: 'id'.

Parameters

  1. attrs

    Angular controller's $attrs.

  2. name (optional)

    String of the first part of random generated id.

Results

  1. Return random generated id or the value of data-fhp-element-id .
