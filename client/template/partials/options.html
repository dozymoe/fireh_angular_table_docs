Options / FhTableDefinition

To setup options, create an instance of FhTableDefinition, and through html markup provide it to table widgets.

Provides empty scope, which primary use is event-hub.

All widgets listen and transmit events by means of this object, eventually getting the overall picture of pieces of data each widget was interested in.

Fields

Main purpose is helper for items.getter, a callback function which will perform ajax requests to the REST api to get list of items.

  • filter
    • throttle

      Delay lookup (ajax requests), will by used by <input type="text"> with popup search hints.

  • filterBy

    Default filter-by for items.getter, a key-value pairs of table-field-name and list of filter-values.

    For example, filterBy: {name: ['john', 'alex'], gender: ['male']}.

  • items
    • getter

      The main work-horse, must return a promise of list of items.

    • identifierFields

      How to identify each items as unique, will be used with _.pick() and _.isEqual().

      Default to identify items by their id field, assuming there was one.

      Required! The widgets will break with a fatal error if you left this out.

    • page

      I have absolutely no idea why this "page" is here, while "filterBy" and "orderBy" are a level above.

      Default active page for items.getter.

    • pageSize

      Determines how many items we want to collect from REST api with a single fetch, will be used with items.getter

  • orderBy

    Default order-by for items.getter, a list of list of two items, table-field-name and 'asc' or 'desc'.

    For example, orderBy: [['id', 'asc'], ['name', 'asc']].

  • eventHandlers

    Over powered event handlers, let you override the behaviour of default widgets.

  • middlewares

    List of names of services which implements eventHandlers.

    It's better that you assigned middlewares with html attribute data-fhp-middlewares on specific widgets, if you placed them here it will act as wildcards to all widget that use this definition.

  • services

    Singletons which life time at least as long as this definition's instance, they provide services, for example to be used as middlewares.

Sub definition fields

Each of these sub definitions are keyed by table-field-names and each table-field-names have the fields above.

  • fieldDefinition
  • filterDefinition
  • orderDefinition
