FhTableFieldSelect

Selection widget for inline form inside the table.

Angular Directive Parameters

  • data-fh-table-field-select itself

    Expected value, string, the field's name.

  • data-fhp-name

    Expected value, string, the field's name, if you haven't set one already with data-fh-table-field-select

  • data-fhp-table

    Expected value, instance of FhTableDefinition, if the widget was outside the scope of FhTableRow but that is very unlikely.

  • data-fhp-template-url

    Expected value, string of url to partial html, with this you can override the template html used by the widget.

    The widget uses Angular $templateRequest on the value.

  • data-fhp-size

    Expected value, integer, the pager size with which to query the server for items.

    You can also set this value when creating instance of FhTableDefinition, the value set with this html attribute will override that one.

  • data-fhp-order-by

    Expected value, string, the order by field-name with which to query the server for items.

    You can also set this value when creating instance of FhTableDefinition, the value set with this html attribute will override that one.

    This method is inferior because you can only order by one field, in FhTableDefinition you can use multiple order-by fields.

  • data-fhp-order-dir

    Expected value, string ('asc' or 'desc'), the order-by's direction.

  • data-fhp-single-selection

    Expected value, string (empty, 'true', or 'false'), an option to disable multiple-selection which is the default.

  • data-fhp-element-id

    Expected value, string, form elements usually have id attribute set, for example to link <label> with the element itself.

    If you don't set one, a random element-id will be assigned.

  • data-fhp-element-name

    Expected value, string, the alphabets part of random element-id, the priority is lower than value set in data-fhp-element-id.

    Random numbers will be appended when the id was constructed, making the id unique.

Demo

See FhTableFieldSelect demo

Notice that data-fhp-editable attribute is added to element with data-fh-table-row, otherwise this widget wouldn't work.

This demo uses FhInfiniteScroll so you can scroll down the list of country flags. Notice that data-fhp-init-callback attribute is added to element with data-fh-infinite-scroll, otherwise the infinite scrolling wouldn't work.

In this demo you'd see the usage of FhTranscluded and FhTranscludePane.

You can freely set the content of the popup-modal in the html itself with angular transclusion. In this particular widget the transclude-pane: 'popup' was built with Bootstraps modals.

Widgets Requirement

  1. FhTable
  2. FhTableRow (editable)
